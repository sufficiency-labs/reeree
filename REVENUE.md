# Revenue Model

## Current Status

reeree is a personal tool being built for own use. Revenue is not a goal for Phase 1-6.

## Potential Future Revenue Streams

### Stream 1: Open Source + Sponsorship
**Source:** GitHub Sponsors, Open Collective
**Pricing:** Voluntary donations
**Target:** Developers who use reeree daily and want to support development
**Expected volume:** $0-500/mo at best. Not a business model — supplemental.

### Stream 2: Premium Features (if ever)
**Source:** Paid add-ons or hosted service
**Pricing:** TBD
**Note:** This is speculative. The core tool must remain free and open source per values. Any premium layer would be convenience (hosted worker pools, team collaboration), never core functionality gated behind payment.

## Market Size

The market for terminal-native LLM coding tools is real and growing (Aider: 4.1M installs, OpenCode: 95K stars, Goose: 27K stars). But reeree's positioning (dispatch console, not chatbot) is novel — no direct comp for market sizing.

## Path to First Dollar

Not a priority. Build for own use. Open source when ready. If it's good, sponsorship/consulting follows naturally. Revenue is a trailing indicator of utility, not a goal.
